\begin{abstract}
Este trabalho investiga as dificuldades metodológicas e a sobrecarga cognitiva enfrentadas por equipes pequenas de software (1--4 membros) ao adotar práticas ágeis inspiradas no Scrum. Propõe-se o desenvolvimento de uma plataforma de gestão ágil autônoma baseada em um sistema multiagente que orquestra um fluxo de trabalho ``com formato de Scrum'' (visão, especificação, roadmap, histórias, planejamento e execução), porém com uma camada adicional de governança por especificação (\emph{Spec-Driven Architecture}). Nessa camada, uma especificação versionada é tratada como fonte primária de verdade para escopo e regras, e a aceitação de artefatos é decidida por \emph{gates} determinísticos de validação (pass/fail), mantendo a geração inicial dos artefatos como não determinística (LLM). A validação é realizada por meio de uma Prova de Conceito (PoC), com coleta de métricas de avaliação como carga cognitiva (NASA-TLX) e eficiência do fluxo (cycle time).
\end{abstract}

\begin{englishabstract}{Autonomous Agile Management Platform with Autonomous Agents: a cognitive approach for small software teams}{Scrum-inspired, multi-agent systems, software engineering, Spec-Driven, cognitive load}
This work investigates the methodological difficulties and cognitive load faced by small software teams (1--4 members) when adopting Scrum-inspired practices. We propose an autonomous agile management platform based on a multi-agent system that orchestrates a Scrum-shaped workflow (vision, specification, roadmap, stories, sprint planning, and execution) while adding a specification-governed layer (Spec-Driven Architecture). In this layer, a versioned technical specification is treated as the primary source of truth for scope and constraints, and artifact acceptance is decided by deterministic validation gates (pass/fail), while artifact generation remains non-deterministic (LLM-based). The proposal is validated through a Proof of Concept (PoC), collecting evaluation metrics such as cognitive load (NASA-TLX) and workflow efficiency (cycle time).
\end{englishabstract}
