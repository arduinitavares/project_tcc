+# Dashboard Architecture Refactor Plan
+
+The current architecture in [main.py](file:///c:/Users/mjnrc/projects/project_tcc/main.py) is procedural and tightly coupled, driving both the CLI event loop and manipulating the FSM and database simultaneously.
+
+To properly integrate the new web **Dashboard** ([frontend/index.html](file:///c:/Users/mjnrc/projects/project_tcc/frontend/index.html) and [api.py](file:///c:/Users/mjnrc/projects/project_tcc/api.py)), we are adopting a **Layered Architecture with the Repository Pattern**, maintaining explicit FSM constraints via a Domain Model.
+
+## Proposed Changes
+
+### 1. Data Source Layer: Repositories
+Extract all direct `sqlite3` and `SQLModel` query logic from the core application into dedicated repository classes.
+
+- [NEW] `repositories/` directory with [__init__.py](file:///c:/Users/mjnrc/projects/project_tcc/db/__init__.py).
+- [NEW] `repositories/product.py`: `ProductRepository` for fetching and updating Products (persisted in [spec_authority_dev.db](file:///c:/Users/mjnrc/projects/project_tcc/db/spec_authority_dev.db)).
+- [NEW] `repositories/session.py`: `WorkflowSessionRepository` for loading and persisting volatile FSM state (currently in [agile_sqlmodel.db](file:///c:/Users/mjnrc/projects/project_tcc/agile_sqlmodel.db)).
+
+### 2. Domain Layer: Explicit FSM Entity
+Ensure the state machine logic is fully decoupled from the event loop and the database.
+
+- [MODIFY] [orchestrator_agent/fsm/controller.py](file:///c:/Users/mjnrc/projects/project_tcc/orchestrator_agent/fsm/controller.py) (or related): Introduce a `WorkflowSession` aggregate root that exposes command methods like `.start()`, `.approve_vision()`, and uses the `FSMController` to validate legal transitions based on the current `OrchestratorState`.
+
+### 3. Application Layer: The Service Envelope
+Break [main.py](file:///c:/Users/mjnrc/projects/project_tcc/main.py)'s synchronous `while True` CLI loop into an asynchronous Application Service that FastAPI can trigger.
+
+- [NEW] `services/` directory with [__init__.py](file:///c:/Users/mjnrc/projects/project_tcc/db/__init__.py).
+- [NEW] `services/workflow.py`: `WorkflowService` that coordinates interactions. For example, a method `trigger_agent_run(session_id, user_input)` that:
+  1. Loads state via `WorkflowSessionRepository`.
+  2. Asks Domain `WorkflowSession` if the transition is allowed.
+  3. Kicks off the LLM Runner.
+  4. Saves the resultant state via `WorkflowSessionRepository`.
+
+### 4. Presentation Layer: API / BFF (Backend-For-Frontend)
+Update the FastAPI app to serve as the JSON Backend-For-Frontend, dropping its placeholder data and wiring it into the new Services and Repositories.
+
+- [MODIFY] [api.py](file:///c:/Users/mjnrc/projects/project_tcc/api.py)
+  - Inject the `ProductRepository` into the `GET /api/projects` endpoint to list real data.
+  - Implement endpoints (e.g., `POST /api/projects/{id}/action`) that call `WorkflowService` methods to actually drive the FSM forward.
+

+

+1.  **Refactor Phase:** Ensure [main.py](file:///c:/Users/mjnrc/projects/project_tcc/main.py) CLI still works when rewritten to use the new `WorkflowService` and Repositories.
+2.  **API Integration Phase:** Verify `uv run api.py` starts successfully.
+3.  **Dashboard Phase:** Open `http://localhost:8000` and confirm the UI projects list is dynamically populated from [spec_authority_dev.db](file:///c:/Users/mjnrc/projects/project_tcc/db/spec_authority_dev.db).
