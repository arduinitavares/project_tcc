## 2024-04-22 - Wrap Modal Inputs in Forms for Accessibility and Usability
**Learning:** Dialogs or modals that contain input fields should always wrap the inputs and submission buttons within a native `<form>` element with a `<button type="submit">`. Relying on generic `div`-based layouts with `onclick` handlers on generic buttons strips away essential browser-native functionality, such as Enter-key submission and HTML5 form validation tooltips (e.g. for `required` fields).
**Action:** When creating or updating modal dialogs with input fields, always wrap the content within a `<form>` element, ensure the submit button has `type="submit"`, cancel/close buttons have `type="button"`, and use an `onsubmit` handler (with `event.preventDefault()`) rather than `onclick` on the button to preserve native HTML5 validation tooltips and standard Enter-key submission behavior.
