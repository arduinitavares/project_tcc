## 2026-05-09 - Native HTML5 Form Validation in Modals
**Learning:** Adding native form validation (via pattern matching and title tooltips) inside interactive modals provides superior accessibility and a smoother workflow. We learned that to preserve both 'Enter to submit' and validation error messages, inputs should be placed inside a `<form>` that blocks the default action (`event.preventDefault()`) and specific buttons (like 'Cancel') must explicitly be set to `type="button"` so they do not inadvertently trigger a submission context.
**Action:** Always wrap modal inputs in a `<form>` element, use native HTML5 properties like `required` and `pattern` to block whitespace-only or empty strings, provide a helpful `title` for the validation tooltip, and ensure non-submitting buttons specify `type="button"`.
